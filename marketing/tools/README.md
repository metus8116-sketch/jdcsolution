# 치과 거리 순위 자동화 도구 (`dentist_rank.py`)

출발지에서 **자동차로 가까운 치과 순위**를 자동으로 뽑고, 특정 치과(기본: 죽전에스치과)가 **몇 등**인지 알려줍니다.
네이버 지도에서 "출발지 → 주변 치과 → 거리순"을 손으로 세던 작업을 자동화합니다.

- 주변 치과 목록·좌표: **카카오 로컬 API** (최대 45곳, 직선거리순 수집)
- 실제 자동차 거리·시간: 두 엔진 선택 (`--mode driving`, 기본값)
  - `--engine osrm` (기본): **무료·키 불필요·승인 불필요** (OpenStreetMap, 실시간 교통 미반영)
  - `--engine kakao`: 카카오내비 길찾기 (실시간 교통 반영, 단 [카카오내비] 활성화/승인 필요)
- 외부 패키지 불필요 (파이썬 표준 라이브러리만 사용)

## 1. 카카오 REST API 키 발급 (무료)
1. https://developers.kakao.com → 로그인 → **내 애플리케이션** 생성
2. **앱 키 → REST API 키** 복사
3. 사용할 API 활성화:
   - **카카오맵(로컬)**: 주변 치과 검색·주소 좌표 변환 (필수)
   - **카카오내비/모빌리티 길찾기**: 자동차 실제 거리 (`driving` 모드 사용 시)
4. 플랫폼 설정에서 사용 환경 등록(웹/서버) 필요할 수 있음

## 2. 설정 & 실행
```bash
export KAKAO_REST_API_KEY="발급받은_REST_키"

# 오산리 양촌마을회관 기준, 죽전에스치과 등수 (자동차 거리)
python3 dentist_rank.py --start "용인시 처인구 모현읍 오산리 양촌마을회관"

# 좌표로 직접 입력 + 반경 8km + 직선거리 모드
python3 dentist_rank.py --start "37.31,127.17" --radius 8000 --mode straight

# 다른 치과 등수 확인
python3 dentist_rank.py --start "모현읍 힐스테이트 몬테로이" --target "오포연세치과"
```

## 3. 옵션
| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--start` | (필수) | 출발지 주소/장소명 또는 `"위도,경도"` |
| `--target` | 죽전에스치과 | 등수를 확인할 치과명 (부분일치) |
| `--keyword` | 치과 | 검색 키워드 |
| `--radius` | 8000 | 검색 반경(m), 최대 20000 |
| `--mode` | driving | `driving`=자동차 실제 거리 / `straight`=직선거리 |
| `--engine` | osrm | driving 계산 엔진: `osrm`(무료·승인불필요) / `kakao`(카카오내비, 승인필요) |
| `--key` | env | 카카오 REST 키 (미지정 시 `KAKAO_REST_API_KEY`) |

## 4. 출력 예시
```
📍 출발지: 양촌마을회관 (경기 용인시 처인구 모현읍 ...)
🦷 반경 8000m 내 치과 23곳 수집 완료

순위 치과명                  자동차거리/시간      주소
------------------------------------------------------------------
  1 모현○○치과              1.2km / 4분        ... 능원리
  ...
  7 죽전에스치과            6.8km / 15분       ... 정든로 5   ⬅ 여기!
==================================================================
✅ '죽전에스치과' 은(는) 23곳 중 자동차 거리 기준 **7등** 입니다.
```

## 참고/주의
- 카카오 키워드 검색은 최대 45곳까지만 수집됩니다(API 제한). 더 넓게 보려면 `--radius` 조정.
- 모빌리티 길찾기 호출량이 많으면 일일 쿼터에 유의(후보 수만큼 호출). 가볍게 보려면 `--mode straight`.
- 순위는 카카오 데이터 기준이며 네이버 지도 결과와 미세하게 다를 수 있습니다(둘 다 도로 경로 기반이라 큰 차이는 없음).

# 등시간 지도 (드라이브타임 맵) — `isochrone.py`

죽전에스치과까지 **차로 10·11·12·13분 이내**에 도달하는 영역을 계산해, **카카오맵 위에 색깔 띠**로 그려줍니다. 광고 타깃 지역을 한눈에 보여주는 자료입니다.

- 드라이브타임: **OSRM**(무료·승인 불필요, 교통량 미반영)으로 격자별 소요시간
- 지역 이름: 카카오 **좌표→행정구역(coord2regioncode)** 으로 "N분 이내 동/읍/리" 목록
- 시각화: 자체 완결형 **iso_map.html** 생성 → **카카오맵 JavaScript SDK**로 표시

## 추가로 필요한 키: 카카오 JavaScript 키
지도 표시에는 REST 키가 아니라 **JavaScript 키**가 필요합니다.
1. 카카오 콘솔 → **앱 설정 → 앱 키 → JavaScript 키** 복사
2. **앱 설정 → 플랫폼 → Web** 에 `http://localhost:8000` (또는 `http://localhost`) 등록

## 실행
```bash
export KAKAO_REST_API_KEY="REST키"
python3 isochrone.py --js-key "JavaScript키"

# 지도 보기 (같은 폴더에서)
python3 -m http.server 8000
# 브라우저: http://localhost:8000/iso_map.html
```

## 주요 옵션
| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--clinic` | (카카오 검색) | 치과 좌표 `"위도,경도"` 직접 지정 |
| `--name` | 죽전에스치과 | 치과 검색어 |
| `--half` | 9.0 | 격자 반경(km) — 13분이 안 잡히면 키우기 |
| `--spacing` | 0.5 | 격자 간격(km) — 작을수록 정밀·느림(0.3 권장 정밀) |
| `--js-key` | env | 카카오 JavaScript 키 (`KAKAO_JS_KEY`) |
| `--no-regions` | off | 행정구역 목록 추출 생략(더 빠름) |
| `--poi` | (없음) | 시간대별로 **더 잘게 쪼갤 시설 키워드**(쉼표로 여러 개). 예: `아파트` / `빌라,타운하우스,마을회관` |
| `--area` | (없음) | **주소에 이 단어가 포함된 곳만** 표시(쉼표로 여러 개). 예: `오포,모현` |

### 동(洞)보다 잘게 — 시설 단위로 쪼개기
```bash
# 아파트 단지 단위
python3 isochrone.py --js-key "JS키" --poi 아파트

# 오포·모현 외곽: 빌라/타운하우스/마을회관만 골라서
python3 isochrone.py --js-key "JS키" --poi 빌라,타운하우스,마을회관 --area 오포,모현
```
→ 콘솔에 **차로 10/11/12/13분 이내 시설 목록**(종류 태그 + 소요분)이 시간대별로 출력되고,
   지도에는 시설마다 **색깔 점**(시간대 색)이 찍히고 우측 **목록 패널**이 나타납니다.
   `--area`로 관심 지역(오포·모현)만 남기면 죽전/수지의 수백 개 아파트에 묻히지 않습니다.
   현수막·엘리베이터·반상회 게시판 등을 "시설 단위"로 집행할 때 타깃 우선순위 자료로 사용하세요.

> 참고: 빌라·타운하우스는 카카오 장소 DB 등록률이 아파트보다 낮아 일부만 잡힐 수 있습니다.
> 마을회관·경로당은 대체로 잘 잡힙니다. 외곽까지 보려면 `--half 10` 처럼 범위를 키우세요.

## 출력
- **콘솔**: 차로 10/11/12/13분 이내에 포함되는 **행정구역 목록 + 면적/셀 수**
- **iso_map.html**: 카카오맵 위 색깔 띠(초록=10분 이내 … 주황=13분) + 범례 + 치과 마커

> 참고: OSRM은 무료 공용 서버라 교통량을 반영하지 않는 **자유속도 추정치**입니다. 띠의 대략적 범위 파악용으로 충분하며, 실측은 ±몇 분 차이가 날 수 있습니다.
